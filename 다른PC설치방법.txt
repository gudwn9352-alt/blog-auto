[1단계] 프로그램 3개 설치 (전부 무료)

전부 "다음 다음 다음 완료" 누르면 됩니다.

1번. Node.js 설치
  → https://nodejs.org 접속
  → 왼쪽 초록 버튼 (LTS) 클릭 → 다운 → 실행 → 다음 다음 완료

2번. Git 설치
  → https://git-scm.com/download/win 접속
  → 자동으로 다운됨 → 실행 → 다음 다음 완료

3번. VS Code 설치
  → https://code.visualstudio.com 접속
  → 파란 버튼 클릭 → 다운 → 실행 → 다음 다음 완료


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[2단계] 코드 가져오기

1. VS Code 열기
2. 키보드에서 Ctrl + ` 누르기 (숫자 1 왼쪽에 있는 키)
   → 아래쪽에 검은 창(터미널)이 열려요
3. 아래 3줄을 한 줄씩 입력하고 Enter:

   git clone https://github.com/gudwn9352-alt/blog-auto.git
   cd blog-auto
   npm install

   ※ npm install은 1~2분 걸려요. 끝날 때까지 기다리세요.

[4단계] 이미지 폴더 복사 (선택)

기존 PC에서 생성한 이미지를 다른 PC에서도 보려면:

1. 기존 PC의 thebada 폴더에서
   public/generated-images/ 폴더를 통째로 복사

2. 새 PC의 blog-auto 폴더 안에
   public/generated-images/ 위치에 붙여넣기

※ 이미지를 새로 생성할 거면 이 단계는 건너뛰어도 됩니다.
※ 원고 텍스트는 Firebase(클라우드)에 저장되어서 자동으로 보입니다.

[코드 최신화하기]

기존 PC에서 수정한 내용을 다른 PC에서 가져오려면:

   git pull
   npm install

이 두 줄만 입력하면 됩니다.

[자주 쓰는 명령어]

프로그램 실행        → npm run dev
프로그램 종료        → Ctrl + C
코드 최신화          → git pull
패키지 재설치        → npm install

[문제 해결]

"Port 3000 is in use" 에러
→ 이미 실행 중이에요. 브라우저에서 localhost:3000 접속

"Module not found" 에러
→ npm install 실행 후 다시 시도

이미지가 안 보여요
→ 기존 PC에서 public/generated-images/ 폴더 복사 필요

구글 드라이브 내보내기 오류
→ 원고 상세에서 "연결 초기화" 클릭 → "내보내기" 클릭 → 재로그인

기타 에러
→ 클로드한테 에러 메시지를 그대로 보여주면 해결해줍니다!

[데이터 저장 위치 안내]

원고 텍스트  → Firebase (클라우드) → 어디서든 자동 동기화
이미지 파일  → public/generated-images/ (로컬) → 수동 복사 필요
브랜드 설정  → Firebase (클라우드) → 어디서든 자동 동기화
API 키      → .env.local (로컬) → 수동 생성 필요
